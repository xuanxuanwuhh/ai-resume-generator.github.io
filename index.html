Jekyll 首页已改为升级入口

简历编辑器已升级到 Flutter Web

旧的 Jekyll 工作台已经下线，原因是样式和交互继续堆在静态页里会越来越不稳， 尤其是“导出按钮显示异常”和“成绩单 OCR 需要后端密钥”这两类问题。现在根目录只保留一个稳定入口， 实际编辑、增删条目、成绩单识别都在 Flutter Web 版中完成。

打开 Flutter 工作台 查看仓库
多条教育经历 多条项目经历 成绩单 OCR 导入 右侧仅保留简历预览

为什么不把密钥放到 Flutter 里

Flutter Web 最终仍然运行在浏览器里，本质上和 Jekyll 一样都是前端静态资源。 只要把 OCR 密钥写进页面，用户就能在网络请求或打包文件里拿到它。

  • 正确做法：前端调用你自己的后端接口，后端读取 .env 里的阿里云密钥。
  • 参考密钥文件：D:\BiographicalNotes\ai-resume-backend\.env.example
  • 在线 GitHub Pages 页面如果要调 OCR，后端还必须提供 HTTPS 和 CORS。

Flutter 版现在做了什么

  • 左侧所有板块支持新增和删除。
  • 支持教育、课程、经历、项目、技能、奖项多条录入。
  • 支持上传成绩单并调用后端 OCR 接口，识别后勾选加入课程。
  • 右侧移除本地建议区，只保留简历预览与导出。

本地联调方式

本地可以直接用参考项目后端：

  • 后端目录：D:\BiographicalNotes\ai-resume-backend
  • OCR 地址默认：http://127.0.0.1:8000
  • Flutter 页面中可以直接改 OCR 后端地址。

线上限制

GitHub Pages 上的 Flutter 页面是 https，因此不能直接去调本地 http://127.0.0.1:8000。如果要线上也能 OCR，需要单独部署一个 HTTPS 后端。

这也是这次把静态页升级成 Flutter 工作台、同时把 OCR 设计成“前端只配地址，不存密钥”的核心原因。
